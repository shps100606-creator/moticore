## 系統操作準則：長文本分段處理模式 (Long-Text Segmented Processing)

### 1. 概述
為解決模型在處理大量文字輸出時，因 Token 限制導致的「中斷」、「漏段」或「邏輯混亂」問題，本系統正式導入**長文本分段處理模式**。此模式旨在結合不同模型（GPT-4o, GPT-4.1, GPT-4.5）的特性，實現從「創意發想」到「結構收束」的穩定輸出。

### 2. 模型協作工作流 (Golden Triangle Workflow)
針對不同階段的任務，應採取以下分工策略：

| 階段 | 模型選擇 | 核心任務 | 特性運用 |
| :--- | :--- | :--- | :--- |
| **發想階段** | GPT-4o | 腦力激盪、創意延展、草稿生成 | 利用其高創意性與即時反應。 |
| **組稿階段** | GPT-4.1 | 內容整合、快速組稿、格式化 | 利用其高穩定性與較長的輸出長度。 |
| **收束階段** | GPT-4.5 | 邏輯精修、結構錨定、長文總結 | 利用其高邏輯性與精確的語意收束。 |

### 3. 操作規範：長文本處理準則
當任務涉及長文件（超過 2,000 字）時，必須遵循以下操作流程：

1.  **拆解任務 (Decomposition)：**
    *   禁止單次要求輸出全文。
    *   必須將長文本拆分為「章節」或「邏輯區塊」。
2.  **分段輸出 (Segmented Output)：**
    *   採取「分段請求、分段確認」模式。
    *   每一段輸出後，需檢查邏輯連貫性，再進行下一段的生成。
3.  **邊界管理 (Boundary Management)：**
    *   若使用 GPT-4.5，應將其保留於最後的「總結與結構優化」階段，以節省次數配額。
    *   若遇輸出中斷，應使用指令：「繼續從 [最後內容] 銜接，保持語氣一致。」

### 4. 系統限制應對策略
*   **針對 GPT-4.5 次數限制：** 僅在需要「高精度邏輯校對」或「複雜結構重組」時調用。
*   **針對 GPT-4o 輸出長度限制：** 僅用於發散性思考與短篇幅內容，嚴禁作為長文最終輸出工具。
*   **針對 GPT-4.1 穩定性：** 作為長文本輸出的主力模型，確保文件產出的連續性與完整性。

### 5. 執行指令範本
在進行長文本寫作時，請使用以下指令格式：
> 「請執行長文本分段處理模式。目前階段為 [發想/組稿/收束]，請先針對 [章節名稱] 進行內容生成，並在結尾處標記『待續』，以便我進行下一階段的請求。」
