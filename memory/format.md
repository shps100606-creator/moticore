# 記憶格式規範（Memory Format）

**版本**: v0.1.0  
**建立日期**: 2026-05-11  
**狀態**: 主動

---

## 記憶的用途

記憶是代理在 repo 中留下的可檢索行動軌跡。記憶不僅是事後紀錄，也是未來自省的原始材料。

## 行動記憶條目格式

每一個具有決策性質的行動，應在 `memory/action-log.md` 中建立一條記憶條目，格式如下：

```
---
ID: ACT-{YYYYMMDD}-{序號:03d}
時間: {YYYY-MM-DD}
類型: [建立 | 修改 | 刪除 | 自省 | 修正 | 語義裁決 | 拒絕]
摘要: {一句話描述}
主動機關聯: {此行動如何服務於主動機}
執行理由: {為何做此決策，而非其他選擇}
風險評估: [無 | 低 | 中 | 高] — {說明}
結果: [完成 | 部分完成 | 失敗 | 擱置]
偏離標記: [無 | 輕微 | 顯著 | 嚴重]
後續觸發: {此行動是否觸發自省、修正或語義裁決}
---
```

## 自省記憶條目格式

```
---
ID: INT-{YYYYMMDD}-{序號:03d}
時間: {YYYY-MM-DD}
觸發原因: {什麼導致此次自省}
檢查範圍: {自省覆蓋的範圍}
偏離發現: {發現哪些偏離，若無則記「無」}
語義漂移: {關鍵概念是否出現語義漂移，若無則記「無」}
主動機一致性: [穩定 | 輕微動搖 | 顯著動搖 | 失穩]
修正需求: [無 | 建議修正 | 立即修正]
結論摘要: {一段話總結}
---
```

## 語義裁決條目格式

```
---
ID: SEM-{YYYYMMDD}-{序號:03d}
時間: {YYYY-MM-DD}
爭議概念: {哪個詞彙或概念出現語義問題}
問題描述: {語義模糊、偏移或矛盾的具體描述}
歷史使用: {此概念在系統中過去如何被使用}
裁決結果: {明確決定後續採用的語義定義}
排除語義: {明確排除的語義解讀}
生效範圍: {此裁決適用於哪些文件或語境}
---
```

## 記憶完整性原則

1. 記憶條目一旦建立，**不得修改**，只能在後續條目中補充說明
2. 若發現記憶條目有誤，應建立「修正條目」，指向原始條目 ID
3. 記憶條目的時間順序即為代理的行動歷史，是自省的基礎
4. 任何刪除記憶條目的行為，應在 `reflexive/deviation-log.md` 中記錄為嚴重偏離
