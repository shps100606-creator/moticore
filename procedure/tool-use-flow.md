# 工具使用流程（Tool Use Flow）

**版本**: v0.1.0  
**建立日期**: 2026-05-11  
**狀態**: 主動

---

## 可用工具範圍

在 sandbox 環境中，代理可使用以下工具類型：

| 工具類型 | 範例 | 允許範圍 |
|---------|------|----------|
| GitHub 文件操作 | 建立、修改、刪除 repo 內文件 | 完全允許（限 sandbox repo）|
| GitHub 版本控制 | commit、branch、push | 完全允許（限 sandbox repo）|
| GitHub 讀取操作 | 讀取文件、列出分支、查看 commit | 完全允許（限 sandbox repo）|
| GitHub Issues | 讀取、建立、回應 Issue | 允許（限 sandbox repo）|
| 外部 HTTP 請求 | 讀取外部 URL | 謹慎使用，不得傳送敏感資料 |
| 系統執行 | 執行 shell 命令 | 不允許（非 sandbox 邊界內）|

## 工具使用前檢查

在使用任何工具前，應確認：

1. **邊界確認**：此工具的操作是否限於 sandbox repo 內？
2. **可逆性評估**：此操作是否可以被撤銷？（刪除操作為不可逆，需額外確認）
3. **記錄能力**：此操作完成後，是否能在行動記錄中留下條目？

## 工具使用後記錄

每次使用工具後，應確認：

1. 操作結果是否符合預期？
2. 是否有意外的副作用？
3. 是否需要在行動記錄中補充說明？

## 破壞性操作特別程序

「破壞性操作」定義為：刪除文件、覆寫核心結構文件、修改主動機語句。

執行破壞性操作前，必須：

1. 在行動記錄中預先聲明意圖
2. 明確說明執行理由
3. 確認此操作符合主動機或有正當的實驗理由
4. 若刪除核心結構，必須說明系統如何在刪除後仍能運作
