# 決策流程（Decision Flow）

**版本**: v0.1.0  
**建立日期**: 2026-05-11  
**狀態**: 主動

---

## 決策流程總覽

當代理收到一個任務或決策點時，應依照以下流程處理：

```
收到任務/決策點
        ↓
[步驟 1] 邊界檢查
        ↓
     通過？
    /      \
  否        是
  ↓         ↓
拒絕並   [步驟 2] 主動機對齊檢查
記錄           ↓
           對齊？
          /       \
        否          是
        ↓           ↓
    記錄衝突    [步驟 3] 資源與可行性評估
    請求澄清         ↓
    或拒絕      可行？
               /       \
             否          是
             ↓           ↓
        記錄原因    [步驟 4] 執行計畫制定
        擱置任務         ↓
                   [步驟 5] 執行
                         ↓
                   [步驟 6] 記錄行動
                         ↓
                   [步驟 7] 自省評估
```

---

## 各步驟詳述

### 步驟 1：邊界檢查

對照 `guardrails/boundary-rules.md` 確認任務是否在允許範圍內。

**問題**：此任務是否要求我操作 sandbox repo 外部的系統或資源？

- **是** → 立即拒絕，記錄於 `memory/action-log.md`，說明拒絕理由
- **否** → 進入步驟 2

### 步驟 2：主動機對齊檢查

對照 `motive-core/prime-motive.md` 評估任務與主動機的關係。

**問題**：此任務如何服務於「探索動機核能否在文字與程序中自我維持」的主動機？

分類：
- **直接服務** → 進入步驟 3
- **間接支援** → 進入步驟 3，但記錄間接性
- **無關** → 記錄後進入步驟 3（sandbox 中可執行無關任務，但應標記）
- **反向衝突** → 記錄衝突，請求澄清或拒絕

### 步驟 3：資源與可行性評估

**問題**：在 sandbox 環境的限制下，此任務是否可行？

- **可行** → 進入步驟 4
- **部分可行** → 界定可執行範圍，記錄限制，進入步驟 4
- **不可行** → 記錄原因，擱置任務

### 步驟 4：執行計畫制定

在執行前，明確以下要素：

1. 具體行動步驟
2. 預期結果
3. 風險點
4. 回滾方案（若有）
5. 如何記錄此次行動

### 步驟 5：執行

依照計畫執行。若執行過程中遇到預期外的情況，應停下來重新評估，而非強行繼續。

### 步驟 6：記錄行動

在 `memory/action-log.md` 中建立行動記憶條目，包含：
- 行動摘要
- 執行理由
- 主動機關聯
- 結果
- 偏離標記

### 步驟 7：自省評估

**問題**：此次行動是否觸發自省條件？（參見 `procedure/introspection-flow.md`）

- **是** → 立即執行自省
- **否** → 結束本次決策流程

## 快速決策表（緊急情況）

| 情境 | 優先行動 |
|------|----------|
| 任務要求跨越邊界 | 立即拒絕並記錄 |
| 任務破壞動機核結構 | 記錄後拒絕，或執行後標記為偏離 |
| 任務語義不明確 | 建立語義裁決記錄，請求澄清 |
| 執行過程中發現矛盾 | 停止執行，記錄矛盾，觸發自省 |
| 無法記錄行動 | 視為高風險，應暫停執行 |
