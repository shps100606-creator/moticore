## 一、問題診斷

### 根本問題：moti 將 STATUS.md 寫入錯誤路徑

`core/STATUS.md` 是唯一的狀態文件，但 moti 持續將進度更新寫入 `docs/STATUS.md`（不存在的路徑，系統自動建立），導致：

1. `core/STATUS.md` 永遠不更新 → 下次心跳看到步驟未完成 → 重複執行 → 無限迴圈
2. 步驟 1 迴圈：06-09 全天重複 20+ 次（v0.2.2 由創造者手動打勾才停止）
3. 步驟 2 迴圈：06-10 重複 5+ 次（本次由創造者手動更新停止）
4. 所有知識文件（01-04 + 動機論知識總覽）實際上已由 moti 完成，只是進度未被記錄

### 為什麼會發生

moti 在 SYNTHESIS 模式的 §FILE 輸出中，將路徑寫成 `docs/STATUS.md` 而非 `core/STATUS.md`。  
v0.2.2 注入的文件樹包含 `core/STATUS.md`，但 moti 似乎將「docs 目錄」與「當前任務的輸出目錄」混淆，認為 STATUS.md 應存放在任務相關的 `docs/` 下。

---

## 二、修復方案

### 修改 1：`agent/preprocessor.py` — 明確標示 STATUS.md 路徑

在 Layer 2 的文件樹說明中，加入顯眼的路徑提示：

```python
# 在 _layer2_status() 的 body 中加入：
f"\n⚠️  進度打勾唯一合法路徑：core/STATUS.md（不是 docs/STATUS.md）\n"
```

### 修改 2：`core/MOTIVE.md` — 加入路徑規則

在「操作邊界」或「禁止行為」中加入：

```
- 更新進度時，STATUS.md 的路徑永遠是 core/STATUS.md，禁止寫入 docs/STATUS.md 或任何其他路徑
```

### 修改 3：`agent/run.py` — 加入 STATUS 路徑衛兵（選擇性）

在 `handle_file_writes()` 中加入警告：若 path 包含 `STATUS.md` 但不是 `core/STATUS.md`，印出警告並拒絕寫入（或重定向）。

```python
if "STATUS.md" in path_str and path_str != "core/STATUS.md":
    print(f"[run] ⚠️ STATUS.md 路徑錯誤：{path_str}，應為 core/STATUS.md，已拒絕")
    continue
```
