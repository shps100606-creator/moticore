# v0.2.2 版本更新計畫書 — 回應架構重整

**日期：** 2026-06-10
**目標版本：** v0.2.2
**前置條件：** v0.2.1 Issue 回覆修復已驗證

---

## 一、問題診斷

### 根本原因

`decision.py` 的 `max_output_tokens=8192` 是硬性上限，但 `REMARKS_INSTRUCTIONS` 不分模式，要求模型每次都準備所有 §SECTION 類型。這導致：

| 模式 | 必要輸出 | 估算 token 用量 | 截斷風險 |
|------|---------|---------------|---------|
| READING | §FILE（筆記 ~2000 chars）+ §FILE（INDEX.md ~1500 chars）+ §ACTION + §READ_REQUEST | ~4000–6000 | 中 |
| SYNTHESIS | §FILE（知識文件，完整論述可達 3000+ chars）× 多份 | ~6000–8000+ | **高** |
| RESPONSE | §ISSUE_RESPONSE × N（每份 ~500 chars）+ §ACTION | ~2000–4000 | 低 |

目前 Issue #32 進入 SYNTHESIS 模式，截斷風險最高。

### 現有架構優勢（未充分利用）

`preprocessor.py` 的 `detect_mode()` 已正確分離三種模式，Layer 3 也按模式給出不同任務。但 `decision.py` 的輸出指令完全不感知模式，等於讓模型帶著 RESPONSE 的包袱去做 READING/SYNTHESIS。

---

## 二、解決方案：Mode-aware 輸出格式

### 核心思路

讓 REMARKS_INSTRUCTIONS 跟著 mode 變化，每種 mode 只要求必要的 §SECTION，釋放 token 空間給真正重要的輸出。

### 各模式輸出規格

**READING 模式：**
```
必填：§ACTION, §FILE（筆記）, §FILE（INDEX.md 更新）, §READ_REQUEST
禁止：§ISSUE_RESPONSE（READING 不回 Issue，改由下一個 RESPONSE 心跳處理）
選填：§QUESTION
```

**SYNTHESIS 模式：**
```
必填：§ACTION, §FILE（知識文件）
禁止：§ISSUE_RESPONSE（同上）
選填：§READ_REQUEST, §QUESTION
注意：§FILE 輸出無字數限制，完整性優先
```

**RESPONSE 模式：**
```
必填：§ACTION, §ISSUE_RESPONSE × N
禁止：§FILE（RESPONSE 不做閱讀/寫文件）
選填：§QUESTION
```

### 輸出 token 調整

測試將 `max_output_tokens` 從 8192 提升至 **16384**（Gemini Flash 系列支援）。Mode-aware 格式 + token 提升雙重保障。

## 三、具體修改項目

### 修改 1：`agent/decision.py`

**`run_consciousness(motive, newspaper)` → `run_consciousness(motive, newspaper, mode)`**

```python
# 依 mode 選擇對應的 REMARKS_INSTRUCTIONS
def _get_instructions(mode: str) -> str:
    if mode == "READING":
        return REMARKS_INSTRUCTIONS_READING
    elif mode == "SYNTHESIS":
        return REMARKS_INSTRUCTIONS_SYNTHESIS
    else:  # RESPONSE
        return REMARKS_INSTRUCTIONS_RESPONSE
```

**`max_output_tokens` 提升：**
```python
max_output_tokens=16384,  # 原 8192
```

**三份 REMARKS_INSTRUCTIONS 常數：**
- `REMARKS_INSTRUCTIONS_READING`：包含 §FILE 筆記格式、§FILE INDEX.md 格式、§READ_REQUEST 格式；明確說明「不輸出 §ISSUE_RESPONSE」
- `REMARKS_INSTRUCTIONS_SYNTHESIS`：包含 §FILE 格式（強調完整性）；明確說明「不輸出 §ISSUE_RESPONSE，不需要 §READ_REQUEST 除非確實需要下一篇原文」
- `REMARKS_INSTRUCTIONS_RESPONSE`：包含 §ISSUE_RESPONSE 格式（引用知識背景）；明確說明「不輸出 §FILE」

### 修改 2：`agent/run.py`

**`call_with_retry` 傳入 mode：**
```python
raw_output = call_with_retry(motive, newspaper, mode=mode, retries=4)
```

**`run_consciousness` 呼叫簽名更新：**
```python
def call_with_retry(motive, newspaper, mode="READING", retries=4):
    ...
    return run_consciousness(motive, newspaper, mode)
```
