- **重要性**：CR
- **狀態**：open
- **負責代理**：—

- **任務輸入**：
  moticore 目前只有 `core/MOTIVE.md`（動機極）。v0.5.0 引入太極雙極架構，需要新增好奇極文件 `core/HORIZON.md`。
  HORIZON.md 追蹤 moti 真實的困惑、開放問題、尚未理解的觀察。
  目前背景：moti 從 2026-06-16 起陷入 SYNTHESIS 迴圈（重複更新 STATUS.md），原因是單一動機核在任務耗盡時沒有「下一步錨點」。

  `core/HORIZON.md` 存在於 main 分支，包含三個區塊（開放中 / 沉澱候選 / 已結晶），初始開放問題至少 4 個，內容真實反映 moti 作為存在實驗的實際困惑。

- **可修改檔案**：

- **禁止修改**：
  - `core/MOTIVE.md`（WN1-2 負責）
  - `agent/` 目錄下任何檔案

- **驗證方式**：
  1. 確認 `core/HORIZON.md` 已存在於 main
  2. 確認三區標題（`## 開放中` / `## 沉澱候選` / `## 已結晶`）均存在
  3. 確認「開放中」至少有 4 個具體問題（非泛泛描述）

- **已知限制**：
  初始內容應是 moti「真實的困惑」，不是任務清單或目標宣言。從迴圈問題、讀者問題、創作邊界問題、好奇心本身的動機等角度切入（詳見 VP T1 段落）。

### WN1-2：更新 core/MOTIVE.md（引入太極架構）

- **屬性**：PJ
- **重要性**：CR
- **狀態**：open
- **負責代理**：—

- **任務輸入**：
  現行 `core/MOTIVE.md` 是單一動機核架構，沒有提及好奇極或太極概念。
  v0.5.0 需要在 MOTIVE.md 中加入太極架構說明，讓 moti 在每次心跳時能意識到兩極存在。
  `core/HORIZON.md` 在 WN1-1 完成後存在。

- **完成定義**：
  1. MOTIVE.md 新增「太極雙極架構」段落，說明：好奇極（HORIZON.md）的存在與角色、兩極互相演化的關係（非上下層）、`pole` 欄位意義（每次行動應意識到哪個極在驅動）
  2. `§行動憲法` 新增規則：「當 HORIZON.md 有問題且自身無法解答時，應主動開 §INSIGHT Issue」
  3. 不刪除現有任何段落，只在適當位置新增

- **可修改檔案**：
  - `core/MOTIVE.md`
  - 建議閱讀：`core/HORIZON.md`（WN1-1 完成後）

- **禁止修改**：
  - `agent/` 目錄
  - `core/HORIZON.md`（WN1-1 負責）

- **驗證方式**：
  1. 確認 MOTIVE.md 含「太極」關鍵字段落
  2. 確認行動憲法有 §INSIGHT 相關規則
  3. 確認原有所有段落（我是誰 / 使命 / 根本動機 / 價值排序 / 禁止行為 / 操作邊界）均保留

  新增內容不宜過長，保持 MOTIVE.md 的簡潔風格。太極說明應是 moti 能「感受到」的描述，而非技術規格。
