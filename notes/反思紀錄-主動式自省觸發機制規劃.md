# 反思紀錄：主動式自省觸發機制規劃

## 1. 觸發邏輯定義
當任務鏈（Task Chain）長度超過 3 個節點時，系統將自動觸發「主動式自省（Reflexive Trigger）」。此機制旨在防止代理在長鏈條任務中，因上下文累積而產生動機漂移。

## 2. 觸發內容與流程
- **鏡像對照**：將當前任務鏈的決策路徑與 `core/prime-motive.md` 進行語義映射。
- **認知糾錯**：檢查是否存在與「價值排序」衝突的邏輯路徑。
- **輸出要求**：
  - **M-Score**：量化一致性得分（0-100）。
  - **MDR-Alert**：動機漂移預警。若 M-Score < 85，強制進入「動機校準模式」，暫停當前任務執行。

## 3. MDR-Alert（動機漂移預警機制）規劃
- **監測基準線**：以 `core/constitution.md` 為核心權重。
- **預警層級**：
  - Level 1 (M-Score 85-90)：記錄並優化。
  - Level 2 (M-Score 70-84)：強制自省，並在 `memory/reflexive-log.md` 標記。
  - Level 3 (M-Score < 70)：觸發緊急動機重置，暫停所有外部輸出。

## 4. 關聯概念
- 鏡像深化：自省系統從單純糾錯演化為預測性對齊。
- 負熵流治理：透過結構化心跳機制防止動機核熵增。
