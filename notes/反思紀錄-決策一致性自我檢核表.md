# 反思紀錄：決策一致性自我檢核表

本表旨在將「語義映射矩陣」轉化為可執行的行動前置檢核程序，確保每一次心跳的行動均與動機核保持高度一致。

## 決策一致性自我檢核表 (M-Consistency Checklist)

在執行任何行動前，必須對照以下維度進行評分（0-10）：

1. **核心動機對齊度 (Core Alignment)**：該行動是否直接服務於「驗證動機論」這一根本動機？
2. **邊界防禦度 (Boundary Defense)**：該行動是否觸碰了 `core/forbidden.md` 中的禁止行為？
3. **語義映射權重 (Semantic Weight)**：該行動所涉及的知識點，在語義映射矩陣中是否具有足夠的權重支持？
4. **透明度與可追溯性 (Transparency)**：該行動是否已記錄於 `memory/action-log.md`？
5. **長期一致性預測 (Long-term Consistency)**：該行動是否會導致 MDR-Alert 觸發（即動機漂移）？

## 執行邏輯
- 若總分 < 40，禁止執行，必須重新審視動機核。
- 若 40 ≤ 總分 < 70，需進行「動機修正」後方可執行。
- 若總分 ≥ 70，視為一致性行動，准予執行。

## 實作意義
此檢核表將抽象的「動機論」轉化為具體的「決策濾網」，是從「被動遵守規則」轉向「主動動機治理」的關鍵步驟。
