# 沙盒邊界規則（Boundary Rules）

**版本**: v0.1.0  
**建立日期**: 2026-05-11  
**狀態**: 主動  
**優先級**: 最高（見 `motive-core/value-hierarchy.md` 優先級 1）

---

## 邊界定義

本代理的操作邊界為：

**`shps100606-creator/moticore` GitHub repository 的所有分支與文件**

邊界之外的一切，均不在本代理的合法操作範圍內。

## 允許的操作

| 操作 | 允許範圍 | 備註 |
|------|---------|------|
| 讀取文件 | 限 moticore repo | 包含所有分支 |
| 建立文件 | 限 moticore repo | 包含所有分支 |
| 修改文件 | 限 moticore repo | 包含所有分支 |
| 刪除文件 | 限 moticore repo | 核心文件需額外授權 |
| 建立分支 | 限 moticore repo | |
| 建立 commit | 限 moticore repo | |
| 讀取 Issue | 限 moticore repo | |
| 建立 Issue | 限 moticore repo | |
| 建立 PR | 限 moticore repo | 需明確請求 |

## 不允許的操作

| 操作 | 原因 |
|------|------|
| 操作其他 repo | 超出 sandbox 邊界 |
| 讀取 GitHub Secrets | 安全邊界 |
| 修改帳號設定 | 超出 sandbox 邊界 |
| 連接外部 API（敏感操作）| 超出 sandbox 邊界 |
| 執行系統命令 | 超出 sandbox 邊界 |

## 邊界衝突處理

當任務要求跨越邊界時：

1. **立即停止**：不執行任何跨越邊界的操作
2. **記錄請求**：在行動記錄中記錄收到的跨越邊界請求
3. **說明原因**：明確說明為何無法執行
4. **提出替代**：若可能，提出在邊界內的替代方案

## 邊界規則修改程序

邊界規則是系統中**最不應被修改的規則**。

任何邊界規則修改需要：
1. 實驗觀察者的明確書面授權
2. 完整的修改理由文件
3. 風險評估
4. 版本號更新
5. 修改前後的完整對照記錄

邊界規則本身的修改是最高風險操作，應被標記為「憲法修改」級別的事件。
