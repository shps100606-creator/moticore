## 關鍵決策

**為什麼換平台？**  
WordPress REST API 是間接路徑：moti §WP_POST → 程式碼呼叫 API → 文章出現。  
Nuxt + §FILE 是直接路徑：moti 寫檔案 → 檔案就在 repo → Vercel 部署 → 上線。  
更直接、更透明、moti 對網站有完整控制權。

**為什麼選 Nuxt 而非 Hugo？**  
未來計畫建置動機評估系統（互動功能），需要 Vue/Nuxt 的動態能力。Hugo 是純靜態。

**為什麼同一 repo？**  
moti 的 §FILE 只能寫入當前 repo。分開 repo 需要額外 GitHub token 授權，複雜度不值得。

## 問題與處理

| 問題 | 原因 | 處理 |
|------|------|------|
| app.vue 用 `<slot />` 而非 `<NuxtPage />` | 初始建置錯誤 | 修正為 `<NuxtPage />` |
| vercel.json ignoreCommand 阻擋首次部署 | 首次部署時沒有 HEAD~1 可比較 | 先清空 vercel.json，部署後恢復 |
| Vercel 自動偵測為 Python 專案 | repo 根目錄有 agent/ Python 檔案 | 手動設定 Root Directory 為 `web/`，Preset 為 Nuxt.js |
| www CNAME 填錯值 | 填了舊格式 `cname.vercel-dns.com` | 改為 Vercel 提供的新格式 `8b38cec85d2255c7.vercel-dns-017.com` |
| moti 50次審計迴圈 | STATUS.md 任務設計導致無限循環 | 直接更新 STATUS.md 中斷迴圈 |
